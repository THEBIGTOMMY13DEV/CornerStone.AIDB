# **Description**

A brief (2-3 sentences) description of the project itself using laymen terms. What is the end product? Cornerstone AI is an evolving academic platform that transforms a student's diagnostic results into a personalized, visual roadmap for their education. The end product is a secure, role-based web application where students receive automated guidance and staff can actively manage classes and assignments through a centralized dashboard.

# **New Technology to Explore**

           A list of potential new technologies, frameworks, libraries, languages etc.. that you will add to your project or leverage in some way. 

* **SQL (PostgreSQL or MySQL):** For structured data storage and relational management.  
* **Node.js & Express:** To build a robust backend server that handles logic and API routes.  
* **Bcrypt.js:** To implement secure password hashing and protection against credential leaks.  
* **Role-Based Access Control (RBAC):** To programmatically manage different permission levels for Students, Staff, and Admins.  
* **CSS Frameworks (e.g., Bulma or Tailwind):** To create a professional, responsive UI for the various user dashboards.

# **Measures of Success**

Clearly define how you will measure your overall success in completion of your capstone artifact. Be as specific as possible, with quantitative measurable. Consider what your team’s M.V.P. is (Minimum Viable Product). The project’s success will be measured by the successful deployment of a functional M.V.P. which includes:

1. Authenticated Data Persistence: A user can create an account, log out, and return to find their data saved exactly as they left it within the SQL database.  
2. Role-Specific Logic: The system must accurately verify a user’s role (Student, Staff, or Admin) upon login.  
3. Quantitative Metric: The system identifies the user's role and redirects them to their own curated dashboard in under 1.5 seconds.  
4. Automated Generation: For students, the platform must successfully generate and store a unique academic roadmap in the backend immediately following the completion of the diagnostic exam.
